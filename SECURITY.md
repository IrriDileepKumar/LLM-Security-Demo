# Security Policy

## ⚠️ IMPORTANT WARNING

**This application is INTENTIONALLY VULNERABLE and designed for educational purposes only.**

This repository contains deliberate security vulnerabilities to demonstrate OWASP Top 10 for Large Language Model Applications. It is designed to help security professionals, developers, and researchers understand LLM security risks in a controlled environment.

## DO NOT USE IN PRODUCTION

- This application contains intentional security flaws
- It is designed to be exploited for educational purposes
- Never deploy this application in a production environment
- Do not use any code patterns from this application in real projects without understanding the security implications

## Intended Use

This application should only be used:
- In isolated, controlled environments
- For security education and training
- For security research purposes
- With explicit permission in penetration testing scenarios

## Security Features Disabled

The following security features are intentionally disabled or misconfigured:
- CORS is set to allow all origins (`*`)
- Debug mode is enabled by default
- Input validation is intentionally weak
- Authentication and authorization are minimal
- Rate limiting is not implemented
- Various injection vulnerabilities are present

## Reporting Security Issues

Since this application is intentionally vulnerable, please DO NOT report the educational vulnerabilities as security issues.

However, if you discover:
- Unintended vulnerabilities that could affect users running the demo
- Security issues in the deployment or setup process
- Vulnerabilities that extend beyond the educational scope

Please report them by opening an issue with the "security" label.
